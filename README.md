## Why?
Why not? 🙃

<em>It all started with a local restaurant I used to order from.</em> 
<em>When I visited their website, I realized how much potential it had — so I decided to rebuild it with a modern, clean design and full ordering functionality.</em> 

## ⚙️ Tech Stack
| Layer | Technologies |
|:------|:--------------|
| **Frontend** | ⚛️ React (Vite), 🎨 Tailwind CSS, 🌍 React Router DOM, 🗣️ i18next (RO/EN localization) |
| **Backend** | 🟢 Express.js, 🔐 JWT Auth with HTTP-only Cookies, 💳 Stripe API |
| **Database** | 🐘 PostgreSQL (hosted via NeonDB), Supabase (for menu-images) |
| **Hosting** | 🚀 Frontend on Vercel, ☁️ Backend on Render, Database on Neon |
| **Other Tools** | 🔄 Framer Motion (animations), 📦 Context API (Cart & Auth), 🧩 REST API integration |

---

## 🍕 Key Features
✅ **User Authentication**  
– Secure register/login system with JWT cookies  
– Session persistence via HTTP-only tokens  

✅ **Dynamic Menu Customization**  
– Choose garnish, salad, sauces, and drinks per menu  
– Real-time total price calculation  

✅ **Smart Cart Management**  
– Orders stored per user/session  
– Edit or delete existing menu items dynamically  

✅ **Checkout Flow**  
– “Finish Order” page with live validation  
– Stripe integration for card payments  
– Auto-move orders from temporary to final tables  

✅ **Responsive Design**  
– Fully optimized for mobile, tablet, and desktop  
– Adaptive layout with smooth animations  

✅ **Multilingual Support**  
– 🌐 English & Romanian interface  
– Saved user preference in localStorage  

✅ **Modern UI Enhancements**  
– Framer Motion transitions  
– Loading skeletons and CircularProgress states  
– Elegant confirmation modals and toasts
